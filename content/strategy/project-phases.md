# Project Phases

## Phase 0: Agent and Content Scaffold

Status: in progress.

Outcomes:

- Add `AGENTS.md` and local agent assets.
- Preserve existing notes.
- Create organized `content/` workspace.
- Define project brief, milestones, phases, and site architecture notes.
- Add the requested frontend-design project skill.

## Phase 1: Content Foundation

Outcomes:

- Convert raw notes into a publishable information architecture.
- Define top-level navigation and content taxonomy.
- Decide which existing drafts become first launch articles.
- Create article briefs for the four approved milestones.
- Identify claims that need external research and citations.

## Phase 2: Website Stack Decision

Outcomes:

- Choose final stack: Next.js plus Docusaurus, Next.js plus MDX, Astro plus Starlight, or another lean static-first option.
- Define hosting, analytics, sitemap, RSS, metadata, and content workflow.
- Create an implementation plan before installing dependencies.

## Phase 3: MVP Website

Outcomes:

- Build minimal landing page.
- Build docs/content hub.
- Publish first set of cornerstone articles.
- Add SEO metadata, sitemap, internal linking, and basic analytics.
- Add a clear path for newsletter, contact, or consulting inquiry if desired.

## Phase 4: Practical Toolkit

Outcomes:

- Add maturity model.
- Add security review checklist.
- Add project AGENTS.md examples.
- Add production-agent architecture templates.
- Add eval, observability, and audit trail templates.

## Phase 5: Production Polish

Outcomes:

- Harden performance, accessibility, metadata, and deploy process.
- Add editorial review workflow.
- Add content update cadence.
- Add visual diagrams and durable evergreen pages.
