# Strategy Directory Guide

## Purpose

`content/strategy/` contains project direction, approved milestones, content planning, phases, and site architecture notes. Treat these files as planning documents, not raw notes or publishable articles.

## Current Files

- `project-brief.md` - mission, audience promise, differentiation, content pillars, tone, and non-goals.
- `milestones.md` - approved milestones and proposed refinements that require user approval.
- `project-phases.md` - phased roadmap from scaffold through production polish.
- `site-architecture.md` - candidate stack options and current website architecture recommendation.
- `content-map.md` - source notes, candidate launch pages, and candidate content series.

## Rules

- Read `project-brief.md` before changing strategy, structure, or content plans.
- Do not revise approved milestones without direct user approval.
- Keep proposed strategy changes clearly marked as proposed until approved.
- Do not treat strategy files as publishable website content.
- Preserve decision history and avoid silently replacing prior rationale.
- If moving or reorganizing strategy content, preserve the original text and leave a clear pointer when practical.
- Update this `AGENTS.md` whenever files are added, removed, renamed, or substantially repurposed in this directory.
